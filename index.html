I test here the github-pages features. The idea is to move my entire ThePythonGameBook project (text AND code examples) from dokuwiki to Github pages.

I'm not sure if this is possible or if it is a good idea. Main advantages as far as i can see them now are:

  • code examples would be hosted in the same system as the book sites ... Github
  • not another password: Collaborators for ThePythonGameBook nearly always already have an account at Github.com.
  • less spammers: spammers and bot-generated accounts are github's problem and no longer my problem (hopefully)
  • multi language versions: a version of ThePythonGameBook in another language (like German) is just another fork.
  • easy to fork and share: by forking the project every Github user can make his own version of ThePythonGameBook...that is exactly what i want.
  • layout:it should be possible to move the whole system to something like Sphinx to make easy html / LaTex / epub export possible
  • comments: getting rid of Discuss as comment system and use Github's version-aware comment system instead
  • mobile: i hope the auto-generated html/css works better on mobile devices than my current dokuwiki roundbox template

    • A part of having no clue if this will work and how to do it i see already some disadvantages:
    • plain html or jerkyll instead of Dokuwiki-syntax: i am used to dokuwiki syntax and very fast with it
    • harder to use: a random user who only wants to correct an typo must now master the editing of html - files via Github.
    • dependent on Github: What if Github decide charge money for hosting ? Can i easy transfer all files to my own server ?
    • ugly: need to learn html tricks to make the layout as pretty as i am used to to from Dokuwiki
    • features: i have no clue at the moment how to include an navigation-tree into the side area. (like dokuwikis indexmenu)

    I will see how this all work out for me. Until i know exactly what i am doing (i don't) i will use this github-pages system for testing.
    First task is to link to other html sites and to include coding pages.
